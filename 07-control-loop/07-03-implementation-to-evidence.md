# Implementation to Evidence

This chapter explains how implementations produce observable signals: tests, telemetry, audits, and artifacts that become evidence. It emphasizes provenance and linkage back to IR.

## Why this matters

Evidence must be anchored to architecture elements or assessments float free of the model.
